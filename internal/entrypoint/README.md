# Package `entrypoint`

**Import path:** `intelligent-lb/internal/entrypoint`

The `entrypoint` package manages HTTP and HTTPS listener ports, each called an *entrypoint*. Every entrypoint is an independently running `http.Server` with its own port binding, middleware chain, and TLS configuration. The design mirrors Traefik's `TCPEntryPoint` system, where each port runs completely independently — one failing does not affect others.

## `EntryPoint` struct

```go
type EntryPoint struct {
    Name    string
    Config  *config.EntryPointConfig
    server  *http.Server
    handler http.Handler
}
```

| Field | Description |
|---|---|
| `Name` | Identifier (e.g., `"web"`, `"websecure"`, `"dashboard"`) |
| `Config` | Address, protocol, optional TLS cert/key paths |
| `server` | The `*http.Server` that listens on the configured address |
| `handler` | The fully wrapped handler (middlewares applied around the base handler) |

---

## `New(name, cfg, handler, middlewares) *EntryPoint`

```go
func New(name string, cfg *config.EntryPointConfig, handler http.Handler,
    middlewares []middleware.Middleware) *EntryPoint
```

Applies the middleware chain around the base handler using `middleware.Chain()`:
```go
if len(middlewares) > 0 {
    chain := middleware.Chain(middlewares...)
    handler = chain(handler)
}
```

Then creates the `http.Server` with production-safe timeouts:
```go
server: &http.Server{
    Addr:         cfg.Address,   // e.g., ":8080" or "0.0.0.0:8443"
    Handler:      handler,
    ReadTimeout:  15 * time.Second,  // read full request (headers + body)
    WriteTimeout: 30 * time.Second,  // write full response
    IdleTimeout:  60 * time.Second,  // keep-alive idle connection lifetime
}
```

### Why These Timeouts?

| Timeout | Value | Protects Against |
|---|---|---|
| `ReadTimeout` | 15s | Slow-loris attacks that trickle request headers slowly |
| `WriteTimeout` | 30s | Connections that never read the response, backpressuring the server |
| `IdleTimeout` | 60s | Long-lived keep-alive connections that consume file descriptors |

## `Start() <-chan error`

```go
func (ep *EntryPoint) Start() <-chan error {
    errCh := make(chan error, 1)
    go func() {
        log.Printf("[ENTRYPOINT] %s starting on %s (protocol: %s)", ep.Name, ep.Config.Address, ep.Config.Protocol)
        var err error
        if ep.Config.Protocol == "https" && ep.Config.TLS != nil {
            certFile, keyFile := ep.Config.TLS.CertFile, ep.Config.TLS.KeyFile
            if certFile == "" { certFile = "server.crt" }
            if keyFile  == "" { keyFile  = "server.key" }
            err = ep.server.ListenAndServeTLS(certFile, keyFile)
        } else {
            err = ep.server.ListenAndServe()
        }
        if err != nil && err != http.ErrServerClosed {
            errCh <- err  // startup failure
        } else {
            errCh <- nil  // normal shutdown
        }
    }()
    return errCh
}
```

The error channel is **buffered** (capacity 1) so the goroutine can always send without blocking, even if no one is receiving on the other end.

`http.ErrServerClosed` is the expected error from `Shutdown()` — it's treated as `nil` (normal termination, not a failure).

**TLS fallbacks**: If `CertFile` or `KeyFile` are empty strings in the config, they default to `"server.crt"` / `"server.key"` in the current working directory. These are typically generated by `tlsutil.GenerateSelfSigned()` at startup.

## `Shutdown(ctx context.Context) error`

```go
func (ep *EntryPoint) Shutdown(ctx context.Context) error {
    log.Printf("[ENTRYPOINT] %s shutting down...", ep.Name)
    if err := ep.server.Shutdown(ctx); err != nil {
        log.Printf("[ENTRYPOINT] %s forced shutdown: %v", ep.Name, err)
        return err
    }
    log.Printf("[ENTRYPOINT] %s shutdown complete", ep.Name)
    return nil
}
```

`http.Server.Shutdown()`:
1. Stops accepting new connections immediately.
2. Closes all idle keep-alive connections.
3. Waits for in-flight requests to complete, respecting the `ctx` deadline.
4. Returns `context.DeadlineExceeded` if requests don't complete before the deadline.

In-flight requests finish naturally — this is the key to zero-downtime shutdown.

## `Manager` struct

```go
type Manager struct {
    mu          sync.RWMutex
    entrypoints map[string]*EntryPoint
    errChans    map[string]<-chan error
}
```

Holds all registered entrypoints and their error channels.

### Methods

| Method | Description |
|---|---|
| `NewManager()` | Creates empty manager |
| `Register(ep)` | Adds an entrypoint (write lock) |
| `Get(name)` | Returns entrypoint by name, or nil (read lock) |
| `Names()` | Returns all registered names (read lock) |
| `StartAll()` | Starts all registered entrypoints |
| `ShutdownAll(ctx)` | Shuts down all concurrently |

### `StartAll()`
```go
func (m *Manager) StartAll() {
    m.mu.Lock(); defer m.mu.Unlock()
    for name, ep := range m.entrypoints {
        m.errChans[name] = ep.Start()  // each runs in its own goroutine
    }
    log.Printf("[ENTRYPOINT] Started %d entrypoints", len(m.entrypoints))
}
```

### `ShutdownAll(ctx context.Context) error`
```go
func (m *Manager) ShutdownAll(ctx context.Context) error {
    m.mu.RLock(); defer m.mu.RUnlock()

    var wg sync.WaitGroup
    var mu sync.Mutex
    var errs []error

    for name, ep := range m.entrypoints {
        wg.Add(1)
        go func(name string, ep *EntryPoint) {
            defer wg.Done()
            if err := ep.Shutdown(ctx); err != nil {
                mu.Lock()
                errs = append(errs, fmt.Errorf("entrypoint %s: %w", name, err))
                mu.Unlock()
            }
        }(name, ep)
    }

    wg.Wait()

    if len(errs) > 0 {
        return fmt.Errorf("shutdown errors: %v", errs)
    }
    log.Printf("[ENTRYPOINT] All entrypoints shut down successfully")
    return nil
}
```

All entrypoints shut down **concurrently** via goroutines + `WaitGroup`. A separate mutex (`mu`) protects the `errs` slice from concurrent appends. Only one shutdown context deadline applies to all — if one entrypoint has a slow request, it doesn't delay others beyond the global deadline.

## `ResolveMiddlewares(names []string, builder *middleware.Builder) ([]middleware.Middleware, error)`

```go
func ResolveMiddlewares(names []string, builder *middleware.Builder) ([]middleware.Middleware, error) {
    var resolved []middleware.Middleware
    for _, name := range names {
        mw, err := builder.Build(name)
        if err != nil {
            return nil, fmt.Errorf("failed to build middleware %q: %w", name, err)
        }
        resolved = append(resolved, mw)
    }
    return resolved, nil
}
```

Converts a list of middleware names (from config) into actual `middleware.Middleware` functions. Called during startup and hot reload for each entrypoint and router.

Fails fast: if any middleware name is unknown, returns an error immediately with the name that failed.

## Complete Lifecycle Example

```go
// 1. Build middleware
builder := middleware.NewBuilder(cfg, svcManager)

// 2. Resolve entrypoint middlewares from config
webMws, _ := entrypoint.ResolveMiddlewares(cfg.EntryPoints["web"].Middlewares, builder)

// 3. Create entrypoints
manager := entrypoint.NewManager()
manager.Register(entrypoint.New("web",       cfg.EntryPoints["web"],       httpRouter,  webMws))
manager.Register(entrypoint.New("websecure", cfg.EntryPoints["websecure"], httpRouter,  webMws))
manager.Register(entrypoint.New("dashboard", cfg.EntryPoints["dashboard"], dashHandler, dashMws))

// 4. Start all
manager.StartAll()

// 5. Wait for OS signal
<-sigCh

// 6. Graceful shutdown
shutdownCtx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
defer cancel()
if err := manager.ShutdownAll(shutdownCtx); err != nil {
    log.Printf("Shutdown errors: %v", err)
}
```

## Config Reference

Entrypoints are configured in `config.json`:
```json
{
  "entryPoints": {
    "web": {
      "address": ":8080",
      "protocol": "http",
      "middlewares": ["headers", "cors", "rate-limit", "access-log", "retry"]
    },
    "websecure": {
      "address": ":8443",
      "protocol": "https",
      "tls": { "cert_file": "server.crt", "key_file": "server.key" },
      "middlewares": ["headers", "cors", "rate-limit", "access-log", "retry"]
    }
  }
}
```
